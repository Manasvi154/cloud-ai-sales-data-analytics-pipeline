PHASE 1 - LOCAL SETUP
==================================================
1) Open terminal in project root:
   c:\Users\kunal\OneDrive\Desktop\Data Automation

2) Create virtual environment:
   python -m venv .venv

3) Activate virtual environment (PowerShell):
   .\.venv\Scripts\Activate.ps1

4) Install dependencies:
   python -m pip install -r requirements.txt

5) Verify interpreter is from .venv:
   python -c "import sys; print(sys.executable)"
   Expected path contains: \Data Automation\.venv\Scripts\python.exe

6) Create environment file:
   copy .env.example .env

7) Edit .env minimum values:
   SECRET_KEY=<strong-random-value>
   AWS_REGION=ap-south-1
   S3_BUCKET_DATASETS=<your-bucket-name>
   AUTH_DB_PATH=instance/data_automation.db
   OTP_EXPIRY_MINUTES=10
   OTP_RESEND_COOLDOWN_SECONDS=45

8) Start app:
   python run.py

9) Open browser:
   http://127.0.0.1:5000

10) Run tests:
    python -m pytest -q

PHASE 3 - AUTH + OTP VALIDATION
==================================================
1) Register a new user:
   - Open /auth/register
   - Submit email + password (>= 8 chars)

2) OTP behavior:
   - If SMTP is configured, OTP arrives by email
   - If SMTP is NOT configured, OTP appears in server logs as:
     [DEV OTP] email=<address> otp=<6-digit-code>

3) Verify OTP:
   - Open /auth/verify
   - Enter email + OTP code
   - Confirm successful verification and auto-login

4) Negative checks:
   - Login before verify should be blocked
   - Wrong OTP should show validation error
   - Resend OTP should respect cooldown

TROUBLESHOOTING
==================================================
1) If PowerShell blocks activation:
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

2) If module import fails:
   Confirm venv is active and rerun:
   python -m pip install -r requirements.txt

3) If AWS auth fails:
   Re-run aws configure and verify IAM credentials from Manasvi.
